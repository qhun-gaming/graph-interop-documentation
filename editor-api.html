Editor API

The Editor API provides visual graph editing capabilities through GraphView. This page covers the editor backend architecture and customization options.

Overview

GraphInterop's editor layer is built on Unity's GraphView framework (part of UI Toolkit), providing a mature, feature-rich visual editing experience while keeping the runtime API separate.

Backend Architecture

GraphInterop uses a backend abstraction to support multiple visual editors:

public interface IGraphEditorBackend { GraphAsset CurrentGraph { get; } void Initialize(EditorWindow window, VisualElement root); void LoadGraph(GraphAsset graph); void OnDisable(); }

Current Backend: GraphViewEditorBackend (GraphView-based) Future Backend: GraphToolkit UI (when mature)

See GraphView Features for detailed GraphView integration.

Main Editor Window

GraphInteropEditorWindow

The base class for graph editors:

using QhunGaming.GraphInterop.Editor; using UnityEditor; public class MyGraphEditor : GraphInteropEditorWindow { [MenuItem("Window/My Graph Editor")] public static void Open() { var window = GetWindow<MyGraphEditor>(); window.titleContent = new GUIContent("My Graph"); window.Show(); } }

Features:

  • Automatic backend selection

  • Asset loading/saving

  • Toolbar (Save, Validate, Execute)

  • Integration with Unity's asset database

Custom Editor Windows

Customize the editor for specific graph types:

public class GameplayGraphEditor : GraphInteropEditorWindow { [MenuItem("Window/Gameplay Graph Editor")] public static void Open() { GetWindow<GameplayGraphEditor>("Gameplay"); } protected override void OnEnable() { base.OnEnable(); // Auto-load last graph var lastPath = EditorPrefs.GetString("LastGameplayGraph", ""); if (!string.IsNullOrEmpty(lastPath)) { var asset = AssetDatabase.LoadAssetAtPath<GraphAsset>(lastPath); if (asset != null) LoadGraph(asset); } } protected override void OnGraphLoaded(GraphAsset asset) { base.OnGraphLoaded(asset); // Save path EditorPrefs.SetString("LastGameplayGraph", AssetDatabase.GetAssetPath(asset)); // Update window title titleContent.text = $"Gameplay - {asset.name}"; } // Filter which nodes appear in search window protected override bool CanCreateNodeType(Type nodeType) { return nodeType.Namespace?.Contains("MyGame.Gameplay") ?? false; } }

Visual Customization

Node Styling

Use NodeInfoAttribute for visual customization:

[NodeInfo( DisplayName = "Damage Calculator", Category = "Combat", Description = "Calculates damage with armor mitigation", Icon = "d_Profiler.Physics", Color = "#FF5733" // Red-orange title bar )] public class DamageNode : GraphNode { }

See GraphView Features for icon reference.

Port Styling

Use PortInfoAttribute for type-specific styling:

[PortInfo( Color = "#4CAF50", // Green ports DisplayName = "Position Data", Icon = "d_Transform Icon" )] public struct PositionData { public Vector3 position; public Quaternion rotation; }

Color Options:

  • Hex: "#RRGGBB"

  • Named: "red", "blue", "green", "orange", "yellow", "purple", "pink"

USS Styling

Customize the editor appearance with USS (Unity Style Sheets):

/* Custom node background */ .node { background-color: rgba(40, 40, 40, 0.8); border-radius: 6px; } /* Port colors by type */ .port[data-type="System.Single"] .connector { background-color: #7fc97f; /* Green for float */ } .port[data-type="UnityEngine.Vector3"] .connector { background-color: #fdc086; /* Orange for Vector3 */ } /* Selected node highlight */ .node.selected { border-color: #FF9800; border-width: 2px; }

USS File Location: Place in Editor/Styles/MyGraphEditor.uss and load in window:

protected override void OnEnable() { base.OnEnable(); var styleSheet = AssetDatabase.LoadAssetAtPath<StyleSheet>( "Assets/Editor/Styles/MyGraphEditor.uss" ); rootVisualElement.styleSheets.Add(styleSheet); }

Backend Registration

Register custom backends:

#if UNITY_EDITOR using QhunGaming.GraphInterop.Editor; using UnityEditor; using UnityEngine.UIElements; public class MyCustomBackend : IGraphEditorBackend { public GraphAsset CurrentGraph { get; private set; } public void Initialize(EditorWindow window, VisualElement root) { // Create custom UI var label = new Label("My Custom Backend"); root.Add(label); } public void LoadGraph(GraphAsset graph) { CurrentGraph = graph; // Update UI } public void OnDisable() { // Cleanup } } // Register backend [InitializeOnLoadMethod] private static void RegisterBackend() { GraphInteropEditorWindow.RegisterBackend(() => new MyCustomBackend()); } #endif

Backend Priority: First registered = default backend

Common Workflows

Creating a New Graph

Create New Graph

  1. Right-click in Project window

  2. Select Create > GraphInterop > Graph Asset

  3. Name the asset

  4. Double-click to open in editor

Adding Nodes

Add Nodes to Graph

  1. Press Space or Right-click in graph

  2. Type to search for node

  3. Press Enter or click to create

  4. Drag to position

Connecting Nodes

Connect Ports

  1. Drag from output port (right side)

  2. Drop on input port (left side)

  3. Compatible ports highlight in green

  4. Incompatible ports show red outline

Testing Execution

Test Graph Execution

  1. Click Validate button to check for errors

  2. Fix any validation errors shown in console

  3. Click Execute button to run with empty context

  4. Check console for Debug.Log output from nodes

Integration with Unity

Asset Database Integration

GraphAsset is a ScriptableObject, fully integrated with Unity's asset database:

// Create asset programmatically var graph = ScriptableObject.CreateInstance<GraphAsset>(); AssetDatabase.CreateAsset(graph, "Assets/MyGraph.asset"); // Add node as sub-asset var node = ScriptableObject.CreateInstance<MyNode>(); AssetDatabase.AddObjectToAsset(node, graph); graph.Nodes.Add(node); // Save EditorUtility.SetDirty(graph); AssetDatabase.SaveAssets();

Double-Click to Open

GraphAssets open in the editor automatically when double-clicked (via [CustomEditor] and OnOpenAsset).

Inspector Integration
