Feature Comparison Overview

Understanding GraphInterop's Position

GraphInterop sits between Unity's GraphView (legacy UI) and GraphToolkit (future official API). This page explains what features are available, where they come from, and what's still in development.

The Three Systems Explained

GraphView

  • What: Unity's UI Toolkit-based graph visualization system

  • Status: Mature but considered legacy

  • Scope: Editor UI only (no runtime execution)

  • Availability: Built into Unity Editor

  • API Stability: Internal API, subject to change

GraphToolkit

  • What: Unity's next-generation graph framework

  • Status: Experimental (0.4.0-exp.2)

  • Scope: Both runtime API and editor UI

  • Availability: Optional package

  • API Stability: Experimental, evolving

GraphInterop

  • What: Interoperability layer between the two

  • Status: Production-ready

  • Scope: Stable runtime API + GraphView editor backend

  • Availability: This package

  • API Stability: Semver-guaranteed

Compatibility Philosophy

Priority Order

When feature conflicts arise between GraphView and GraphToolkit, GraphInterop follows this priority:

  1. GraphToolkit API Design (primary)

    • We adopt GTK's API patterns and conventions

    • Future Unity direction takes precedence

  2. Stable Public API (critical)

    • Breaking changes require major version bump

    • Migration guides provided

  3. GraphView Compatibility (secondary)

    • We may break legacy GraphView patterns to align with GTK

    • Editor experience improved where possible

Backward Compatibility Statement

Important: GraphInterop's public API (namespaces starting with QhunGaming.GraphInterop) is stable and follows semver. However, we do NOT guarantee backward compatibility with:

  • Direct GraphView API usage

  • Internal Unity API patterns

  • Experimental GraphToolkit features

Always use the GraphInterop abstraction layer to future-proof your code.
