Getting Started with GraphInterop

TL;DR

GraphInterop is a Unity package that provides a unified node-graph API compatible with both Unity's GraphView and GraphToolkit. It enables you to create visual node-based tools and runtime graph execution systems with a stable, semantic-versioned API.

What is GraphInterop?

GraphInterop is an interoperability layer that:

  • âœ… Adapts Unity's GraphToolkit API for use with GraphView

  • âœ… Provides a stable, semver-compatible public API

  • âœ… Enables runtime graph execution with async and Burst support

  • âœ… Works with Unity 2021.3+

Key Philosophy: GraphInterop aims to be feature-compatible with GraphToolkit's API while supporting GraphView in the editor. As GraphToolkit is Unity's intended successor to GraphView, we prioritize GTK's features, which may require breaking changes from legacy GraphView patterns.

Quick Start (5 Minutes)

1. Installation

Add the package to your Unity project via Package Manager:

https://github.com/qhun-gaming/graph-interop.git

Or add to manifest.json:

{ "dependencies": { "de.qhun.gaming.graphinterop": "https://github.com/qhun-gaming/graph-interop.git" } }

2. Create Your First Node

using QhunGaming.GraphInterop; [NodeInfo( DisplayName = "Add Numbers", Category = "Math", Description = "Adds two numbers together" )] public class AddNode : GraphNode { protected override void OnDefinePorts(IPortDefinitionContext context) { context.AddInputPort<float>("a") .WithDisplayName("A") .WithDefaultValue(0f) .Build(); context.AddInputPort<float>("b") .WithDisplayName("B") .WithDefaultValue(0f) .Build(); context.AddOutputPort<float>("result") .WithDisplayName("Result") .Build(); } public override void Process(GraphContext context) { var a = this.GetInputValue<float>("a", context); var b = this.GetInputValue<float>("b", context); this.SetOutputValue("result", a + b, context); } }

3. Create a Graph Asset

Right-click in the Project window:

Create > GraphInterop > Graph Asset

4. Open the Graph Editor

Double-click the graph asset, or go to:

Window > GraphInterop > Graph Editor

5. Add Nodes and Execute

  • Space or Right-click to open the node search menu

  • Add your AddNode and connect ports

  • Click Execute in the toolbar to test

Feature Highlights

Type-Safe Port System

// Automatic type compatibility checking context.AddInputPort<Transform>("target").Build(); context.AddOutputPort<GameObject>("object").Build(); // âŒ Cannot connect - types incompatible context.AddInputPort<float>("value").Build(); context.AddOutputPort<int>("count").Build(); // âœ… Can connect - numeric types auto-convert

Async Node Support

public class LoadAssetNode : GraphNode, IAsyncNode { public async Task ProcessAsync(GraphContext context) { var address = this.GetInputValue<string>("address", context); var handle = Addressables.LoadAssetAsync<GameObject>(address); await handle.Task; this.SetOutputValue("result", handle.Result, context); } }

Burst-Compiled Nodes

public class BurstMathNode : GraphNode, IBurstNode { private NativeArray<float> _input, _output; public JobHandle ScheduleBurstJob(GraphContext context) { _input[0] = this.GetInputValue<float>("value", context); var job = new MathJob { Input = _input, Output = _output }; return job.Schedule(); } public void CompleteBurstJob(GraphContext context) { this.SetOutputValue("result", _output[0], context); _input.Dispose(); _output.Dispose(); } }

Portal System (Clean Long Connections)

Key Insight: Your custom nodes only depend on the stable GraphInterop API. The backend (GraphView/GTK) can change without breaking your code.

API Stability Guarantee

GraphInterop follows semantic versioning:

  • Major version (1.x.x → 2.0.0): Breaking API changes

  • Minor version (1.0.x → 1.1.0): New features, backward-compatible

  • Patch version (1.0.0 → 1.0.1): Bug fixes

Breaking Changes Policy:

  • We may break backward compatibility with GraphView to align with GraphToolkit

  • We will never break the public GraphInterop API without a major version bump

  • Migration guides provided for all breaking changes
