GraphExecutor

Executes a graph by processing nodes in topological order, resolving connections automatically.

Class Definition

namespace QhunGaming.GraphInterop { public class GraphExecutor { public void Execute(GraphAsset graph, GraphContext context); } }

Execution Process

1. Topological Sort

Nodes are sorted so that dependencies (connected inputs) are processed before dependents:

// If B depends on A: // A.output → B.input // Then A is processed before B

Algorithm: Depth-first search with cycle detection

2. Input Resolution

For each input port on the current node:

Scenario

Action

Connected

Get cached output value from source port

Not connected + Required

Use embedded value (or warn if null)

Not connected + Optional

Keep default/embedded value

3. Node Processing

Calls node.Process(context) which reads from input fields and writes to output fields.

4. Output Caching

After processing, all output port values are cached using the key format:

{nodeId}.{portName}

Usage

Basic Execution

var executor = new GraphExecutor(); var context = new GraphContext(); executor.Execute(graphAsset, context);

With Context Data

var context = new GraphContext(); context.Set("spline", mySplineContainer); context.Set("terrain", myTerrain); executor.Execute(graphAsset, context);

Error Handling

Errors during node processing are caught and re-thrown with additional context:

try { node.Process(context); } catch (Exception ex) { Debug.LogError($"Error executing node '{node.GetType().Name}' (ID: {node.Id}): {ex.Message}"); throw; }

Data Flow Example

Internal cache after execution:

node1.output = 5 node2.output = 2 node3.result = 10

Topological Sort Details

The sort ensures correct execution order:

private List<GraphNode> TopologicalSort(GraphAsset graph) { // Build dependency map: node → nodes it depends on // DFS from each unvisited node // Add to result list in post-order }

Cycle Detection: If a node is visited while still in the recursion stack, a cycle exists.

Performance Considerations

  • O(V + E) time complexity for topological sort

  • Port values are cached in a dictionary (O(1) lookup)

  • Reflection for port access is cached per node instance

Thread Safety

Not thread-safe. The executor should only be called from the main thread due to:

  • Unity API usage in nodes

  • Shared cache dictionary

  • No synchronization primitives

Best Practices

  1. Reuse executor instances - no internal state between executions

  2. Validate before executing - call graph.Validate() first

  3. Handle exceptions - wrap in try-catch for graceful failure

  4. Use context for shared data - don't rely on static state

var errors = graph.Validate(); if (errors.Count > 0) { foreach (var e in errors) Debug.LogError(e); return; } try { executor.Execute(graph, context); } catch (Exception ex) { Debug.LogError($"Graph execution failed: {ex}"); }
