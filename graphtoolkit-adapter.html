GraphToolkit Adapter

The GraphToolkit adapter provides the same functionality as GraphView but uses Unity's experimental Graph Toolkit (GTK) as the backend.

Note: GraphToolkit is experimental and only available in Unity 6.2+

Compilation

The adapter is conditionally compiled when both defines are present:

#if USING_GTK && UNITY_6000_2_OR_NEWER // GraphToolkit implementation #endif

Architecture Difference

GraphToolkit uses a Model-View separation unlike GraphView:

InteropGraphModel

Maps GraphAsset to GTK's GraphModel:

public class InteropGraphModel : GraphModel { private GraphAsset _graphAsset; public void LoadFromAsset(GraphAsset asset) { _graphAsset = asset; // Clear existing DeleteElements(NodeModels); DeleteElements(EdgeModels); // Create node models foreach (var node in asset.Nodes) { var nodeModel = CreateNode<InteropNodeModel>(); nodeModel.Initialize(node); nodeModel.Position = node.EditorPosition; } // Create edge models foreach (var connection in asset.Connections) { // Find source/target ports and create edge } } public override IEnumerable<PortModel> GetCompatiblePorts(PortModel startPort) { // Type compatibility check using TypeCompatibility class } }

InteropNodeModel

Wraps GraphNode for GTK:

public class InteropNodeModel : NodeModel { private GraphNode _node; public void Initialize(GraphNode node) { _node = node; Title = GetDisplayName(node); // Create ports from attributes foreach (var portDef in node.GetInputPorts()) { AddInputPort(portDef.DisplayName, portDef.DataType); } foreach (var portDef in node.GetOutputPorts()) { AddOutputPort(portDef.DisplayName, portDef.DataType); } } // Embedded value support public override bool HasEmbeddedValue(PortModel port) { var portDef = GetPortDefinition(port); return portDef?.ShowEmbedded ?? false; } }

InteropPortModel

Custom port with type compatibility:

public class InteropPortModel : PortModel { public Type DataType { get; set; } public PortDefinition Definition { get; set; } public bool IsRequired { get; set; } public override bool IsCompatibleWith(PortModel other) { if (other is InteropPortModel otherPort) { return TypeCompatibility.AreCompatible( Direction == Direction.Output ? DataType : otherPort.DataType, Direction == Direction.Input ? DataType : otherPort.DataType); } return false; } }

Command Pattern

GTK uses commands for undo/redo support:

public class CreateInteropNodeCommand : Command { public Type NodeType { get; } public Vector2 Position { get; } public override void Execute(GraphModel graphModel) { var node = Activator.CreateInstance(NodeType) as GraphNode; var nodeModel = graphModel.CreateNode<InteropNodeModel>(); nodeModel.Initialize(node); nodeModel.Position = Position; // Also add to GraphAsset ((InteropGraphModel)graphModel).GraphAsset.AddNode(node); } }
