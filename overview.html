GraphInterop Help

GraphInterop Overview

GraphInterop is an interoperability layer that provides a unified API for building node-based graph editors in Unity. It abstracts away the differences between Unity's GraphView API (available in all Unity versions) and the newer GraphToolkit API (Unity 6.2+).

Purpose

The library enables developers to:

  • Define graph nodes using a simple attribute-based approach

  • Serialize graphs as ScriptableObjects

  • Execute graphs at runtime with automatic dependency resolution

  • Use either GraphView or GraphToolkit as the visual backend without changing application code

Key Features

  • Backend Agnostic: Same runtime code works with GraphView or GraphToolkit

  • Attribute-Based Ports: Define inputs/outputs with [Input] and [Output] attributes

  • Type-Safe Connections: Automatic type compatibility checking between ports

  • Topological Execution: Graphs execute in dependency order automatically

  • Blackboard Variables: Shared variables accessible across all nodes

  • Embedded Values: Input ports can have inline editors when unconnected

Quick Start

  1. Create a custom node by extending GraphNode:

[NodeInfo(Category = "Math", Title = "Add")] public class AddNode : GraphNode { [Input] public float a; [Input] public float b; [Output] public float result; public override void Process(GraphContext context) { result = a + b; } }

  1. Create a custom GraphAsset:

[CreateAssetMenu(menuName = "My Graph")] public class MyGraphAsset : GraphAsset { }

  1. Execute the graph:

var executor = new GraphExecutor(); var context = new GraphContext(); executor.Execute(myGraphAsset, context);
