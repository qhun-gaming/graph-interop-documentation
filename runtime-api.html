Runtime API

The runtime API provides all classes needed to define, serialize, and execute node graphs without any editor dependencies.

Overview

The runtime API follows Unity's GraphToolkit design patterns, providing a stable foundation for graph-based systems that works in both editor and builds.

Node Definition Pattern

GraphInterop uses the Builder Pattern (GraphToolkit-compatible) for defining nodes:

using QhunGaming.GraphInterop; using UnityEngine; [NodeInfo( DisplayName = "Add Numbers", Category = "Math", Description = "Adds two numbers together" )] public class AddNode : GraphNode { protected override void OnDefinePorts(IPortDefinitionContext context) { context.AddInputPort<float>("a") .WithDisplayName("A") .WithDefaultValue(0f) .Build(); context.AddInputPort<float>("b") .WithDisplayName("B") .WithDefaultValue(0f) .Build(); context.AddOutputPort<float>("result") .WithDisplayName("Result") .Build(); } public override void Process(GraphContext context) { var a = this.GetInputValue<float>("a", context); var b = this.GetInputValue<float>("b", context); this.SetOutputValue("result", a + b, context); } }

Key Features:

  • Fluent port configuration via builders

  • Type-safe generic methods

  • Default values and display names

  • Lazy port initialization

Builder Interfaces

Port Definition Context

public interface IPortDefinitionContext { // Generic (type-safe) IInputPortBuilder<T> AddInputPort<T>(string name); IOutputPortBuilder<T> AddOutputPort<T>(string name); // Non-generic (runtime type) IInputPortBuilder AddInputPort(string name); IOutputPortBuilder AddOutputPort(string name); }

Input Port Builder

public interface IInputPortBuilder<T> { IInputPortBuilder<T> WithDisplayName(string displayName); IInputPortBuilder<T> WithDefaultValue(T defaultValue); IInputPortBuilder<T> Required(); void Build(); }

Output Port Builder

public interface IOutputPortBuilder<T> { IOutputPortBuilder<T> WithDisplayName(string displayName); void Build(); }

Option Builder

public interface IOptionBuilder<T> { IOptionBuilder<T> WithDisplayName(string displayName); IOptionBuilder<T> WithDefaultValue(T defaultValue); IOptionBuilder<T> WithTooltip(string tooltip); void Build(); }

Node Options

Options are configuration values that affect node topology:

[NodeInfo(DisplayName = "Multi Output", Category = "Utility")] public class MultiOutputNode : GraphNode { protected override void OnDefineOptions(IOptionDefinitionContext context) { context.AddOption<int>("outputCount") .WithDisplayName("Output Count") .WithDefaultValue(3) .Build(); } protected override void OnDefinePorts(IPortDefinitionContext context) { context.AddInputPort<float>("input") .WithDisplayName("Input") .Build(); // Dynamic ports based on option int count = this.GetOptionValue<int>("outputCount"); for (int i = 0; i < count; i++) { context.AddOutputPort<float>($"output{i}") .WithDisplayName($"Output {i}") .Build(); } } public override void Process(GraphContext context) { var input = this.GetInputValue<float>("input", context); int count = this.GetOptionValue<int>("outputCount"); for (int i = 0; i < count; i++) { this.SetOutputValue($"output{i}", input * (i + 1), context); } } }

Option Use Cases:

  • Dynamic port count

  • Operation mode selection (enum)

  • Configuration affecting structure

Graph Execution

Basic Execution

using QhunGaming.GraphInterop; using UnityEngine; public class GraphRunner : MonoBehaviour { public GraphAsset graphAsset; private async void Start() { // Create context var context = new GraphContext(); // Execute graph var executor = new GraphExecutor(graphAsset); await executor.ExecuteAsync(context); } }

With Context Variables

public class GameplayGraphRunner : MonoBehaviour { public GraphAsset graphAsset; private async void Update() { var context = new GraphContext(); // Provide runtime context variables context.SetContextVariable("transform", transform); context.SetContextVariable("deltaTime", Time.deltaTime); var executor = new GraphExecutor(graphAsset); await executor.ExecuteAsync(context); } }

Data Flow

Value Resolution Priority

When GraphExecutor resolves input values, it uses this priority:

  1. Connected Value (from source node's output)

  2. Embedded Value (user-set in editor)

  3. Default Value (defined in builder)

// Example input resolution protected override void OnDefinePorts(IPortDefinitionContext context) { context.AddInputPort<float>("multiplier") .WithDisplayName("Multiplier") .WithDefaultValue(1.0f) // ← Fallback (priority 3) .Build(); } // In editor, user can set embedded value (priority 2) // Or connect another node's output (priority 1)

Setting and Getting Values

public override void Process(GraphContext context) { // Get input values var input = this.GetInputValue<float>("input", context); var multiplier = this.GetInputValue<float>("multiplier", context); // Compute var result = input * multiplier; // Set output values this.SetOutputValue("result", result, context); }

Async and Burst Support

Async Nodes

For I/O operations, web requests, Addressables:

using System.Threading.Tasks; public class LoadAssetNode : GraphNode, IAsyncNode { protected override void OnDefinePorts(IPortDefinitionContext context) { context.AddInputPort<string>("address") .WithDisplayName("Address") .Required() .Build(); context.AddOutputPort<GameObject>("asset") .WithDisplayName("Asset") .Build(); } public async Task ProcessAsync(GraphContext context) { var address = this.GetInputValue<string>("address", context); var handle = Addressables.LoadAssetAsync<GameObject>(address); await handle.Task; this.SetOutputValue("asset", handle.Result, context); } public override void Process(GraphContext context) { throw new System.NotImplementedException("Use ProcessAsync"); } }

Burst Nodes

For high-performance SIMD operations:

using Unity.Burst; using Unity.Collections; using Unity.Jobs; public class BurstNormalizeNode : GraphNode, IBurstNode { private NativeArray<Vector3> _input, _output; protected override void OnDefinePorts(IPortDefinitionContext context) { context.AddInputPort<Vector3>("vector").Build(); context.AddOutputPort<Vector3>("normalized").Build(); } public JobHandle ScheduleBurstJob(GraphContext context) { _input = new NativeArray<Vector3>(1, Allocator.TempJob); _output = new NativeArray<Vector3>(1, Allocator.TempJob); _input[0] = this.GetInputValue<Vector3>("vector", context); var job = new NormalizeJob { Input = _input, Output = _output }; return job.Schedule(); } public void CompleteBurstJob(GraphContext context) { this.SetOutputValue("normalized", _output[0], context); _input.Dispose(); _output.Dispose(); } public override void Process(GraphContext context) { throw new System.NotImplementedException("Use Burst job"); } [BurstCompile] private struct NormalizeJob : IJob { [ReadOnly] public NativeArray<Vector3> Input; public NativeArray<Vector3> Output; public void Execute() { Output[0] = Input[0].normalized; } } }

Validation

Graph Validation

var errors = graphAsset.Validate(); if (errors.Count > 0) { foreach (var error in errors) Debug.LogError(error); return; // Don't execute invalid graph } // Execute await executor.ExecuteAsync(context);

Custom Node Validation

public override void Validate(List<string> errors) { base.Validate(errors); // Always call base first var path = this.GetOptionValue<string>("filePath"); if (string.IsNullOrEmpty(path)) { errors.Add($"Node '{this.Id}': File path is required"); } else if (!System.IO.File.Exists(path)) { errors.Add($"Node '{this.Id}': File not found: {path}"); } }

Attributes

NodeInfo Attribute

Provides metadata for editor and node search:

[NodeInfo( DisplayName = "Calculate Damage", Category = "Combat/Damage", Description = "Calculates damage with armor mitigation", Icon = "d_Profiler.Physics", Color = "#FF5733" )] public class DamageNode : GraphNode { }

Properties:

  • DisplayName: Node title in editor

  • Category: Hierarchical category for search window

  • Description: Tooltip/subtitle

  • Icon: Unity builtin icon name

  • Color: Hex color for title bar

PortInfo Attribute

Customizes port appearance by type:

[PortInfo( Color = "#4CAF50", DisplayName = "World Transform", Icon = "d_Transform Icon" )] public struct TransformData { public Vector3 position; public Quaternion rotation; }

Best Practices

1. Use Generic Builders

// ✅ Good: Type-safe context.AddInputPort<Transform>("target").Build(); // ❌ Avoid: Requires runtime type context.AddInputPort("target") .WithDataType(typeof(Transform)) .Build();

2. Provide Default Values

// ✅ Good: Sensible fallback context.AddInputPort<float>("multiplier") .WithDefaultValue(1.0f) .Build(); // ❌ Risky: May be null/default context.AddInputPort<float>("multiplier").Build();

3. Mark Required Inputs

// ✅ Good: Validates before execution context.AddInputPort<Transform>("target") .Required() .Build();

4. Use Options for Topology

// ✅ Good: Affects structure context.AddOption<int>("portCount").Build(); // ❌ Bad: Should be input port context.AddOption<float>("speed").Build(); // Runtime data!

5. Handle Null Values

public override void Process(GraphContext context) { var target = this.GetInputValue<Transform>("target", context); if (target == null) { Debug.LogWarning("Target is null"); return; // Graceful failure } // Process... }
