TypeCompatibility

Static utility class for checking and scoring type compatibility between ports.

Class Definition

namespace QhunGaming.GraphInterop { public static class TypeCompatibility { public static bool AreCompatible(Type outputType, Type inputType); public static int GetCompatibilityScore(Type outputType, Type inputType); public static bool IsNumericType(Type type); public static bool HasConverter(Type from, Type to); public static void RegisterConverter<TFrom, TTo>(Func<TFrom, TTo> converter); public static object Convert(object value, Type targetType); } }

Compatibility Rules

Types are compatible if any of these conditions are met:

1. Exact Match

// float → float ✓ AreCompatible(typeof(float), typeof(float)) // true

2. Object Type

// Anything → object ✓ AreCompatible(typeof(string), typeof(object)) // true // object → Anything ✓ (runtime checked) AreCompatible(typeof(object), typeof(int)) // true

3. Inheritance / Interfaces

// Derived → Base ✓ AreCompatible(typeof(List<int>), typeof(IEnumerable<int>)) // true

4. Numeric Conversions

// int → float ✓ AreCompatible(typeof(int), typeof(float)) // true // double → int ✓ (may lose precision) AreCompatible(typeof(double), typeof(int)) // true

5. Custom Converters

TypeCompatibility.RegisterConverter<Vector3, float>(v => v.magnitude); AreCompatible(typeof(Vector3), typeof(float)) // true

int score = TypeCompatibility.GetCompatibilityScore( typeof(List<int>), typeof(IEnumerable<int>) ); // Returns ~85 (inheritance)

Numeric Types

The following types are considered numeric:

  • int, uint

  • float, double, decimal

  • long, ulong

  • short, ushort

  • byte, sbyte

TypeCompatibility.IsNumericType(typeof(float)) // true TypeCompatibility.IsNumericType(typeof(string)) // false

Custom Converters

Registering Converters

// Register Vector3 → float converter TypeCompatibility.RegisterConverter<Vector3, float>(v => v.magnitude); // Register string → int converter TypeCompatibility.RegisterConverter<string, int>(s => int.Parse(s));

Using Converters

var vector = new Vector3(3, 4, 0); var magnitude = (float)TypeCompatibility.Convert(vector, typeof(float)); // magnitude = 5.0f

Usage in Editor

The InteropGraphView uses compatibility checking when connecting ports:

public override List<Port> GetCompatiblePorts(Port startPort, NodeAdapter nodeAdapter) { // ... if (TypeCompatibility.AreCompatible(outputType, inputType)) { compatiblePorts.Add(port); } // ... }

Best Practices

Do

  • Register converters for domain-specific types early (e.g., during editor initialization)

  • Use inheritance for flexible port types

  • Prefer specific types over object for type safety

Don't

  • Rely on implicit numeric conversions for precision-sensitive data

  • Register multiple converters for the same type pair (last one wins)

Example: Spline Pipeline Types

// Initialize converters for spline types public static class SplineConverters { [InitializeOnLoadMethod] static void RegisterConverters() { // ISplineProvider → SplineContainer TypeCompatibility.RegisterConverter<ISplineProvider, SplineContainer>( provider => provider.GetSplineContainer() ); // float3 → Vector3 TypeCompatibility.RegisterConverter<float3, Vector3>( f => new Vector3(f.x, f.y, f.z) ); } }

Thread Safety

All static methods are thread-safe for reading. Converter registration should be done during initialization before any parallel access.
